# The Agent Stack Is Fragmenting — And That's the Signal

Every week I track who's actually building toward autonomous AI systems.

Here's what W12 tells me:

**The "agent framework" market is splitting into 3 lanes:**

1. **Orchestration-first** (CrewAI, LangGraph) — multi-agent coordination, enterprise workflows
2. **Code-native** (Anthropic Claude Code, OpenAI Codex, Devin) — agents that write and ship code autonomously
3. **Vertical specialists** — agents built for one domain (legal, finance, healthcare) that outperform generalists 10x

The mistake most people make: treating this like a "which framework wins" race.

It's not. It's a stack formation.

Just like web dev settled into frontend/backend/infra, AI agents are settling into orchestration/execution/domain layers.

**What this means if you're building:**

→ Don't pick a framework. Pick a layer.
→ Orchestration is commoditizing fast. The value is in vertical execution.
→ The teams shipping real autonomous workflows (not demos) are combining 2-3 tools, not betting on one.

**Real signal this week:**
- Enterprise adoption of multi-agent systems hit mainstream discussion at every major AI conference
- The gap between "agent demo" and "agent in production" remains the #1 unsolved problem
- Reliability and evaluation frameworks are becoming the bottleneck, not model capability
